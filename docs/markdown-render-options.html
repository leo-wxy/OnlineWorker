Session Markdown 渲染风格选择

这个页面只用于视觉选择，不改 App 代码。四个方案都修正了当前截图里的主要问题： 行内代码只显示为小胶囊，只有真正的 fenced code block 才显示为代码块。

比较 inline code 比较代码块 比较列表/引用/表格 比较聊天气泡密度

A. Calm Prose

推荐默认。接近 ChatGPT/Linear 的干净阅读感，代码视觉克制，适合长回复。

Recommended

Markdown 渲染判断

TG 日志里的 最终回复已按 HTML edit 是 Telegram API 的格式化方式，不代表 App Session Tab 使用 HTML 注入。

App 侧仍应使用 react-markdown。关键是把 displayMode: "markdown" 的内容交给正确的 Markdown 组件渲染。

判断标准

  • 粗体 应该直接加粗，而不是显示 **粗体**。
  • 列表应该有正常缩进和项目符号。
  • inline code 应该是行内小胶囊，不应该变成整行黑块。
如果粗体、列表、代码块都已经呈现为样式化内容，这轮验证可以判定通过。
function normalizeDisplayMode(role) {
  return role === "assistant" ? "markdown" : "plain";
}
项目期望
inline code行内、小面积、低干扰
code block块级、可横向滚动

选择 A：最稳，适合直接作为默认 Session Markdown 样式。
