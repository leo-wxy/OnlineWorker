<implementation_requirements>
16-02 should provide Claude external hook ingress:

- Claude plugin owns merge/install/uninstall/dedupe of global
  `~/.claude/settings.json` hook entries.
- OnlineWorker hook entries must be marker-based and idempotent.
- Uninstall/disable removes only OnlineWorker-owned marker entries.
- Malformed settings must return a failed/degraded result without destroying
  the user's settings file.
- Lifecycle hooks should map into the existing normalized event chain:
  - `SessionStart` -> `session.created`
  - `UserPromptSubmit` -> `message.user.submitted` and `turn.started`
  - `Stop` / `SessionEnd` -> `message.assistant.final` and `turn.completed`
  - provider error payload -> `turn.failed`
- Existing approval/question hook behavior must remain intact.
- Delta/token streaming is not required for 16-02.
</implementation_requirements>
