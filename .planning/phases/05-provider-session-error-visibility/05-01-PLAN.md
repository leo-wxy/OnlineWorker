# 05-01: Add Provider Session Async Error Normalization and Session Browser Visibility

## Goal

Make asynchronous provider failures visible through the existing generic Session Browser read path, without adding provider-specific React branches.

## Problem Evidence

- External overlay provider Session Browser send returned successfully from the provider send path.
- The upstream model call later failed with `403 insufficient_quota`.
- The provider persisted an assistant message containing an `error` object and no text parts.
- OnlineWorker read normalization filtered that empty assistant record, so polling never observed a new visible assistant turn.

## Implementation Steps

1. Add external overlay provider storage coverage for assistant messages that contain `data.error` but no text parts.
2. Convert provider assistant error records into a provider-neutral assistant turn:
   - `role: assistant`
   - `text: <visible error>`
   - `displayMode: plain`
   - `kind: error`
3. Update fallback provider session bridge normalization to preserve visible error turns and `displayMode/kind` metadata.
4. Update provider owner bridge normalization with the same rules, because packaged App reads generic provider sessions through the owner bridge when available.
5. Keep empty non-error assistant placeholders filtered.
