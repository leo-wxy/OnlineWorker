# Phase 20 Plan 01 — Diagnostics And Support Bundle Backend

## Goal

Add Tauri-owned bounded diagnostics, shared redaction, generated support artifacts, local ZIP creation, and unit tests.

## Tasks

1. Add `commands/support_bundle.rs` with serializable report/check/bundle result contracts and independent bounded checks.
2. Reuse existing config, service, Dashboard/provider, socket, plugin, version, and log truth through narrow shared helpers rather than parallel rules.
3. Add deterministic redaction for credentials, authorization headers, Telegram bot URLs, home paths, and sensitive config values.
4. Generate only documented text/JSON/YAML/log/manifest artifacts in a temporary staging directory; cap the log input at 2 MiB.
5. Use native macOS save and archive facilities to create the ZIP and native Finder reveal for the returned path.
6. Register `run_support_diagnostics`, `export_support_bundle`, and `reveal_support_bundle` Tauri commands.
7. Add Rust tests for healthy/partial reports, timeouts, missing/invalid inputs, redaction, forbidden files/content, manifest, and export failure.

## Verification

- `cargo fmt --manifest-path mac-app/src-tauri/Cargo.toml -- --check`
- focused Rust support-bundle tests
- full Rust tests outside sandbox when socket/process tests require it
- `git diff --check`

## Must Not

- Do not read provider transcript/session-history stores.
- Do not copy `.env` or raw `config.yaml`.
- Do not restart services, mutate config, or perform network requests.
