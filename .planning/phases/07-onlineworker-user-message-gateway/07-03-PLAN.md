# 07-03: Add Codex Remote App-Server User Message Proxy

## Objective

Add a Codex CLI path that can rewrite user text before it reaches Codex app-server, using the existing OnlineWorker user-message gateway and neutralizer.

## Scope

- Probe the real `codex --remote` WebSocket protocol.
- Confirm whether initial CLI prompt text appears in app-server JSON-RPC traffic.
- Add a production in-process WebSocket proxy for OnlineWorker-managed Codex TUI host sessions.
- Route `turn/start` and `turn/steer` text input through `core/user_messages/gateway.py`.
- Fail closed when the proxy cannot start, so the TUI host does not silently bypass the gateway.
- Preserve text items with `text_elements` by skipping rewrite to avoid corrupting UI byte ranges.

## Acceptance Criteria

- Real `codex --remote ws://...` traffic shows initial prompt text in `turn/start.params.input`.
- A sensitive text prompt is rewritten before app-server emits `item/started` for the `userMessage`.
- The local Codex transcript stores only the rewritten prompt text for the probed turn.
- Production proxy uses the existing `prepare_user_message_text` gateway rather than duplicating normalization rules.
- `message_hooks.enabled=false` and builtin hook `mode=off` disable the rewrite.
- Inputs with non-empty `text_elements` are left unchanged.
- TUI host startup fails closed if the proxy cannot be created.
