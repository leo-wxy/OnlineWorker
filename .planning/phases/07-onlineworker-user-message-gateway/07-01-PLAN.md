# 07-01: Add OnlineWorker User Message Gateway and Before-Send Hooks

## Objective

Introduce one OnlineWorker-level gateway for provider-bound user messages and add a first `before_user_message_send` hook that conservatively normalizes abusive wording while preserving user intent.

## Architecture

Add `core/user_messages/` as the shared product-level message boundary. Input adapters construct a common request, the gateway runs OnlineWorker-level hooks, and existing provider `message_hooks` remain responsible for provider-specific connection, thread, and send behavior.

## Scope

- Add a request/result contract for provider-bound user messages.
- Add a configurable `before_user_message_send` hook pipeline.
- Add deterministic `abusive_language_normalization` as the first built-in hook.
- Route existing OnlineWorker-managed user send paths through the gateway:
  - Telegram thread messages.
  - Session Tab / owner bridge sends.
  - provider session bridge sends.
  - new-thread initial messages.
- Investigate and, if supported by Codex hook return protocol, route Codex CLI `UserPromptSubmit` through the same gateway.

## Out of Scope

- LLM-based rewriting.
- UI settings for hook configuration.
- Rewriting provider output, notifications, approvals, final replies, or attachment file contents.
- Changing provider-specific hook contracts beyond passing gateway-processed text into the current provider send sequence.
