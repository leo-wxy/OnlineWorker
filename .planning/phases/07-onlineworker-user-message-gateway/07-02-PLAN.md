# 07-02: Add Dictionary-Backed User Message Neutralizer

## Objective

Replace the first hard-coded abusive-language replacement implementation with a small, maintainable, dictionary-backed neutralizer that can be exercised directly from a script.

## Scope

- Add a pure Python sensitive-term model and matcher.
- Add a neutralizer that supports `drop` and `replace` actions.
- Preserve fenced code block behavior.
- Keep slash-command and command-dispatch skipping in the existing hook runner.
- Add a script for manual normalization checks.
