# 14-03 Plan: Close Consumer Boundary Cleanup

**Status:** source verified
**Created:** 2026-06-05
**Phase:** 14. Unified Message Event Bus
**Trigger:** Phase 14 design-review decisions after 14-02 source verification

## Problem

Phase 14 had two remaining cleanup items after the TaskBoard activity stream
hardening:

- Notification completed-summary logic still lived behind old
  `core.notifications.result_summary` helper names even though the runtime path
  already called `MessageEventBus.notification_summary`.
- TaskBoard still carried a hidden/remove-from-board state and command even
  though the target UI model is pin/unpin only.

Both issues kept compatibility code around after the target Phase 14 boundary
was decided, making future debugging harder.

## Goal

Close the Phase 14 consumer boundary cleanup so:

- Notification completion summary has one bus-consumer implementation boundary.
- TaskBoard card management has only pin/unpin, with no hidden state, command,
  UI affordance, type field, or i18n key.

## Scope

In scope:

- Move completed notification summary local/AI fallback logic under
  `core/messages/notification_summary.py`.
- Remove the old completed-summary helper names and imports from
  `core/notifications/result_summary.py`.
- Remove TaskBoard hidden state and `hide_task_board_session`.
- Remove the TaskBoard remove-from-board button, props, command type, model
  filtering, TypeScript declarations, locale entries, and tests using hidden
  state.
- Add/keep regression checks that prevent hidden/remove command strings from
  returning to the TaskBoard source.

Out of scope:

- App Session detail live rendering migration.
- Telegram rendering/topic migration.
- Approval/question command-boundary implementation.
- Packaged-app verification unless explicitly approved in the current
  conversation.

## Implementation Summary

Completed on 2026-06-05:

- `core/messages/notification_summary.py` now owns completed-summary extraction,
  local fallback title/body selection, AI scenario invocation, and fallback
  message creation.
- `core/notifications/result_summary.py` now contains only reusable notification
  text/title helpers. The old completed-summary helper names were removed.
- `TaskBoardState` now persists only `version` and `pinned`.
- `hide_task_board_session` was removed from the Tauri command module and
  command registration.
- `TaskBoard.tsx` removed the remove-from-board button and only calls
  `pin_task_board_session` / `unpin_task_board_session`.
- `buildTaskBoardModel` no longer reads hidden refs or filters pinned cards by
  hidden state.
- `TaskBoardState` / `TaskBoardTask` frontend declarations no longer expose
  hidden fields.
- TaskBoard locale contract and locale files no longer define
  `removeFromBoard`.
- Focused tests were updated to use pin/unpin-only TaskBoard state and assert
  that hidden/remove command strings are absent from the TaskBoard page.
