# 10-01: Audit Structure Debt and Establish Staged Refactor Rails

## Objective

Create a concrete structure-debt inventory and verification baseline for Phase 10, then select the first safe implementation slices for behavior-preserving refactors.

## Scope

- Rank oversized and high-coupling modules by responsibility count, blast radius, and test coverage.
- Identify extraction seams for Rust/Tauri commands, Python bot/runtime modules, provider runtime modules, and React pages/components.
- Define the characterization test matrix that must pass before and after each refactor slice.
- Update codebase planning docs with the current structure risks and the staged refactor order.
- Produce a summary that names the first implementation slice and why it is safe to start there.

## Non-Goals

- No production code movement in this plan.
- No public contract/schema changes.
- No packaged-app build unless the selected next slice touches startup, IPC, or packaged assets.
- No deleting compatibility code or historical provider behavior.

## Current High-Priority Candidates

1. `mac-app/src-tauri/src/commands/config_provider.rs`
   - 2800 lines.
   - Mixes provider metadata, notification metadata, AI config documents, embedded assets, config IO, and normalization.
   - Existing `config_provider/ai_config_store.rs` shows a safe local extraction pattern.

2. `mac-app/src-tauri/src/commands/dashboard.rs`
   - 2136 lines.
   - Mixes provider config snapshots, provider runtime status IPC, config readiness, CLI checks, Codex/Claude recent activity, state aggregation, and tests.
   - Has internal tests and pure helper seams suitable for first Rust extraction.

3. `bot/events.py`
   - 1785 lines.
   - Mixes event normalization, streaming message editing, approval/question UI, notification delivery, AI summary invocation, topic materialization, and provider-specific sync.
   - High blast radius; should not be first code movement without additional characterization.

4. `bot/handlers/workspace.py`
   - 1240 lines.
   - Mixes workspace scanning, topic naming, callback identity, history replay, thread open, and CLI callback handling.
   - Pure helpers for topic names/callback tokens/history batches are a good Python extraction candidate.

5. `mac-app/src/pages/Dashboard.tsx`
   - 696 lines.
   - Mixes dashboard data loading, derived health state, actions, and presentation.
   - Existing hooks/components pattern makes hook extraction low-risk after backend refactor baseline.
