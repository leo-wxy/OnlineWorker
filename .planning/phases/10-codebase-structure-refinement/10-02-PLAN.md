# 10-02: Extract Tauri Config and Dashboard Helper Modules

## Objective

Reduce the size and mixed responsibilities of `config_provider.rs` and `dashboard.rs` by extracting pure helper modules while preserving all existing Tauri command behavior.

## Scope

- Move embedded provider/notification asset access and metadata helpers out of `config_provider.rs`.
- Move config document provider parsing helpers out of `config_provider.rs` where practical.
- Move dashboard provider status/health helpers out of `dashboard.rs`.
- Move dashboard recent activity readers/cache helpers out of `dashboard.rs` only if the split is mechanical and tests remain focused.
- Keep all `#[tauri::command]` functions and public response structs compatible.

## Non-Goals

- No behavior changes to provider metadata, notification metadata, AI config, dashboard status, or recent activity.
- No Rust schema or frontend contract changes.
- No sidecar startup or service supervision refactor.
- No provider-specific Codex/Claude session parsing refactor in this slice.
