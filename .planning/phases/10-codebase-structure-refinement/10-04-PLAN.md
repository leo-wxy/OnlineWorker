# Plan 10-04: Extract Frontend Dashboard State and Presentation

## Goal

Finish the low-risk Phase 10 frontend structure slice by reducing
`mac-app/src/pages/Dashboard.tsx` from a mixed data/action/presentation module
into a page orchestration layer backed by dashboard-specific components and
state helpers.

## Scope

- Move Dashboard presentation sections into `mac-app/src/components/dashboard/`.
- Move pure Dashboard view-model helpers into a dashboard component-local model
  module.
- Extend `useDashboardState` to expose derived provider/control/open-host state
  so pages do not recalculate it inline.
- Keep existing visual layout, text, Tauri command names, and command payloads
  unchanged.

## Non-Goals

- Do not change backend dashboard command response shapes.
- Do not change service start/stop/restart behavior.
- Do not redesign the Dashboard UI.
- Do not touch app startup, sidecar packaging, bridge IPC, or installed-app data
  paths.
- Do not run packaged-app verification for this slice.

## Tasks

1. Extract Dashboard pure helpers and view model.
   - [x] Move health styles, alert resolution, provider descriptions, provider
     accents, epoch-age formatting, service-control status, provider list, and
     Codex TUI host openability helpers into a dashboard model module.
2. Extract Dashboard presentation components.
   - [x] Add Dashboard hero, sidebar, alert, error, provider status list,
     provider icon, Telegram badge, and setting switch components.
3. Reduce `Dashboard.tsx` to orchestration.
   - [x] Keep Tauri invokes and callback wiring in the page.
   - [x] Delegate all major presentation sections to dashboard components.
4. Extend `useDashboardState`.
   - [x] Expose providers, service control status, and Codex TUI host
     openability as derived state.
5. Verify behavior-preserving frontend checks.
   - [x] Run dashboard/app shell Node tests.
   - [x] Run TypeScript check.
   - [x] Run `git diff --check`.

## Verification

Required:

```bash
node --test mac-app/tests/appShell.test.mjs mac-app/tests/dashboardProviderStatus.test.mjs
cd mac-app && ./node_modules/.bin/tsc --noEmit
git diff --check
```

Packaged-app verification is not required because this slice only moves React
presentation and hook helper code behind the same Tauri invoke calls.
