# Sketch Manifest

## Design Direction

Phase 19 reuses the existing Task Board Tab but may redesign its internal information architecture. The board should make actionable provider-backed states obvious without turning the desktop app into a decorative dashboard. Session lifecycle actions must stay close to the real Session they affect.

## Reference Points

- Existing OnlineWorker navigation shell and Task Board data model
- Linear/GitHub-style restrained desktop information density
